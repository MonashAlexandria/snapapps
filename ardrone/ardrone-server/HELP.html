How to setup an AR-Drone server

Step 1

Connect to the drone's wireless network. Your computer will need to have wireless functionality. This may take some doing. If you just cannot connect to the drone, try holding down the reset button on the bottom for a few seconds before trying again.

Step 2

Extract the entire .ZIP archive to any directory

Step 3

If you're using Windows: Run "START.bat". Either double click it, or run it using command prompt.
If you're using Mac: Open a Terminal window, cd into the ardrone-server directory and run "sh START.sh".

Step 4

By this point a server should be running. If you want computers over the network to be able to control the drone, you may need to setup port-forwarding to allow other computers to access this one. Contact the network administrator if you can't or don't know how to do this.
N.B: If you are using the version 1 AR-Drone, you will receive errors regarding the video stream:
TcpVideoStream error: connect ECONNREFUSED
Attempting to reconnect to TcpVideoStream...
Don't worry! This is normal for the version 1 drone, since the library where the server is from does not fully support the version 1.

Step 5

Use the special AR-Drone version of "Snap!" to program the drone!
To connect to the server:
  1. Select the stage in the bottom right
  2. Select the "Ardrone" block set in the top left
  3. Drag in the "Connect to ardrone-webflight server with hostname []" block
  4. Write the hostname in, or leave it as default if the server is running on the same computer.
  5. Click the block just once to initiate the connection
  6. Drag in the "Connection status" block
  7. Click the "Connection status" block to test the connection. If it says "Connection Succeeded!", you're ready to use the other blocks!
